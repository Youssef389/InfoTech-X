Module 1: Introduction to SQL

1.1 What is SQL?

SQL (Structured Query Language) is a standard language for managing and manipulating relational databases...

-- SQL Example: Creating a table
CREATE TABLE Users (
    UserID INT PRIMARY KEY,
    Username VARCHAR(50),
    Password VARCHAR(50)
);

1.2 SQL Database Management Systems (DBMS)

SQL can be used with various DBMSs such as MySQL, PostgreSQL, SQLite, and SQL Server...

Module 2: Basic Queries

2.1 SELECT Statement

The SELECT statement is used to retrieve data from a database...

-- SQL Example: Selecting data
SELECT * FROM Users;

2.2 Filtering Results

Use the WHERE clause to filter records based on specific conditions...

-- SQL Example: Filtering results
SELECT * FROM Users
WHERE Username = 'john_doe';

Module 3: Data Manipulation

3.1 INSERT INTO Statement

The INSERT INTO statement is used to add new rows to a table...

-- SQL Example: Inserting data
INSERT INTO Users (UserID, Username, Password)
VALUES (1, 'john_doe', 'password123');

3.2 UPDATE Statement

The UPDATE statement is used to modify existing data...

-- SQL Example: Updating data
UPDATE Users
SET Password = 'newpassword456'
WHERE Username = 'john_doe';

3.3 DELETE Statement

The DELETE statement is used to remove rows from a table...

-- SQL Example: Deleting data
DELETE FROM Users
WHERE Username = 'john_doe';

Module 4: Joins

4.1 INNER JOIN

INNER JOIN returns records with matching values in both tables...

-- SQL Example: Inner join
SELECT Orders.OrderID, Users.Username
FROM Orders
INNER JOIN Users ON Orders.UserID = Users.UserID;

4.2 LEFT JOIN

LEFT JOIN returns all records from the left table and matched records from the right table...

-- SQL Example: Left join
SELECT Users.Username, Orders.OrderID
FROM Users
LEFT JOIN Orders ON Users.UserID = Orders.UserID;

Module 5: Subqueries

5.1 Basic Subqueries

Subqueries are queries nested inside other queries...

-- SQL Example: Subquery
SELECT Username
FROM Users
WHERE UserID IN (SELECT UserID FROM Orders WHERE OrderID = 101);

5.2 Correlated Subqueries

Correlated subqueries reference columns from the outer query...

-- SQL Example: Correlated subquery
SELECT Username
FROM Users u
WHERE EXISTS (SELECT * FROM Orders o WHERE o.UserID = u.UserID AND o.OrderID = 101);

Module 6: Indexes & Views

6.1 Indexes

Indexes are used to improve query performance...

-- SQL Example: Creating an index
CREATE INDEX idx_username ON Users (Username);

6.2 Views

Views are virtual tables based on the result of a query...

-- SQL Example: Creating a view
CREATE VIEW UserOrders AS
SELECT Users.Username, Orders.OrderID
FROM Users
INNER JOIN Orders ON Users.UserID = Orders.UserID;

Module 7: Transactions

7.1 Transactions Overview

Transactions are used to execute a series of SQL operations as a single unit...

7.2 COMMIT and ROLLBACK

COMMIT saves all changes made during the transaction, while ROLLBACK undoes them...

-- SQL Example: Transaction
BEGIN;
UPDATE Users SET Password = 'finalpassword' WHERE Username = 'john_doe';
COMMIT;

Module 8: Stored Procedures

8.1 What are Stored Procedures?

Stored procedures are precompiled SQL statements that can be executed as needed...

-- SQL Example: Creating a stored procedure
CREATE PROCEDURE GetUserByUsername(IN username VARCHAR(50))
BEGIN
    SELECT * FROM Users WHERE Username = username;
END;

Module 2: Data Types & Variables

2.1 Basic Data Types

C# provides a variety of data types, including int, float, double, char, and string...

int age = 30;
float height = 5.9f;
double pi = 3.141592653589793;
char initial = 'A';
string name = "John Doe";

Console.WriteLine($"Age: {age}, Height: {height}, Pi: {pi}, Initial: {initial}, Name: {name}");

2.2 Variables and Constants

Variables store data, while constants store data that doesn't change...

const double Gravity = 9.81;

int x = 10;
int y = 20;
int z = x + y;

Console.WriteLine($"z = {z}, Gravity = {Gravity}");

Module 3: Control Structures

3.1 Conditional Statements

C# supports if, else, and switch statements...

int number = 10;

if (number % 2 == 0)
{
    Console.WriteLine("Even number");
}
else
{
    Console.WriteLine("Odd number");
}

switch (number)
{
    case 10:
        Console.WriteLine("Number is ten");
        break;
    default:
        Console.WriteLine("Number is not ten");
        break;
}

3.2 Loops

Loops allow repeated execution of code blocks using for, while, and do-while...

for (int i = 0; i < 5; i++)
{
    Console.WriteLine($"For loop iteration {i}");
}

int j = 0;
while (j < 5)
{
    Console.WriteLine($"While loop iteration {j}");
    j++;
}

int k = 0;
do
{
    Console.WriteLine($"Do-While loop iteration {k}");
    k++;
} while (k < 5);

Module 4: Object-Oriented Programming

4.1 Classes and Objects

Object-Oriented Programming (OOP) in C# involves creating classes and objects...

class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public void Greet()
    {
        Console.WriteLine($"Hello, my name is {Name} and I am {Age} years old.");
    }
}

Person person = new Person();
person.Name = "John";
person.Age = 30;
person.Greet();

4.2 Inheritance

4.2 Inheritance

Inheritance allows a class to inherit fields and methods from another class...

class Animal
{
    public void Eat()
    {
        Console.WriteLine("Eating...");
    }
}

class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine("Barking...");
    }
}

Dog dog = new Dog();
dog.Eat();  // Inherited method
dog.Bark(); // Method from Dog class

4.3 Polymorphism

Polymorphism allows methods to have different implementations depending on the object...

class Shape
{
    public virtual void Draw()
    {
        Console.WriteLine("Drawing a shape");
    }
}

class Circle : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a circle");
    }
}

Shape shape = new Circle();
shape.Draw();  // Outputs: Drawing a circle

4.4 Encapsulation

Encapsulation involves hiding the internal state of an object and requiring all interactions to occur through methods...

class BankAccount
{
    private double balance;

    public void Deposit(double amount)
    {
        balance += amount;
    }

    public double GetBalance()
    {
        return balance;
    }
}

BankAccount account = new BankAccount();
account.Deposit(100);
Console.WriteLine($"Balance: {account.GetBalance()}");

5.1 Arrays

Lists in C# are dynamic arrays that can grow in size...

Module 6: Delegates & Events

6.1 Delegates

Delegates are type-safe pointers to methods...

delegate void PrintDelegate(string message);

class Program
{
    static void PrintMessage(string message)
    {
        Console.WriteLine(message);
    }

    static void Main()
    {
        PrintDelegate del = PrintMessage;
        del("Hello, Delegate!");
    }
}

6.2 Events

Events provide a way for a class to notify other classes when something happens...

class Publisher
{
    public event Action OnPublish;

    public void Publish()
    {
        OnPublish?.Invoke();
    }
}

class Subscriber
{
    public void Subscribe(Publisher publisher)
    {
        publisher.OnPublish += () => Console.WriteLine("Event received");
    }
}

Publisher pub = new Publisher();
Subscriber sub = new Subscriber();
sub.Subscribe(pub);

pub.Publish();

Module 7: Exception Handling

7.1 Try-Catch

Exception handling in C# is done using try and catch blocks...

try
{
    int a = 10;
    int b = 0;
    int result = a / b;
}
catch (DivideByZeroException ex)
{
    Console.WriteLine("Cannot divide by zero.");
}

7.2 Finally

The finally block is used to execute code regardless of whether an exception is thrown...

try
{
    // Some code
}
catch (Exception ex)
{
    // Handle exception
}
finally
{
    Console.WriteLine("This will always execute.");
}

7.3 Custom Exceptions

You can create custom exceptions by extending the Exception class...

class MyCustomException : Exception
{
    public MyCustomException(string message) : base(message)
    {
    }
}

try
{
    throw new MyCustomException("This is a custom exception.");
}
catch (MyCustomException ex)
{
    Console.WriteLine(ex.Message);
}

Module 9: LINQ (Language Integrated Query)

9.1 Introduction to LINQ

LINQ is a powerful feature in C# that provides querying capabilities for collections...

int[] numbers = { 1, 2, 3, 4, 5 };

var evenNumbers = from n in numbers
                  where n % 2 == 0
                  select n;

foreach (var num in evenNumbers)
{
    Console.WriteLine(num);
}

9.2 LINQ with Objects

LINQ can be used to query any object that implements IEnumerable...

List<string> names = new List<string> { "Alice", "Bob", "Charlie" };

var shortNames = names.Where(n => n.Length <= 3);

foreach (var name in shortNames)
{
    Console.WriteLine(name);
}

9.3 LINQ to SQL

LINQ can be used to query databases directly...

var query = from cust in db.Customers
            where cust.City == "London"
            select cust;

foreach (var customer in query)
{
    Console.WriteLine(customer.Name);
}

Module 10: Async Programming

10.1 Introduction to Async

Asynchronous programming in C# allows tasks to run in the background without blocking the main thread...

async Task<string> GetDataAsync()
{
    await Task.Delay(1000); // Simulate a delay
    return "Data retrieved";
}

async void Main()
{
    string data = await GetDataAsync();
    Console.WriteLine(data);
}

10.2 Using async and await

The async keyword is used to define an asynchronous method, and await is used to wait for the completion of an asynchronous task...

async Task PerformTasksAsync()
{
    Task task1 = Task.Delay(2000);
    Task task2 = Task.Delay(1000);

    await Task.WhenAll(task1, task2);

    Console.WriteLine("Both tasks completed.");
}

async void Main()
{
    await PerformTasksAsync();
}

10.3 Async Best Practices

Best practices for async programming include avoiding blocking calls, using ConfigureAwait(false) when appropriate, and handling exceptions properly...

async Task ProcessDataAsync()
{
    try
    {
        await SomeOperationAsync();
    }
    catch (Exception ex)
    {
        Console.WriteLine($"Error: {ex.Message}");
    }
}
